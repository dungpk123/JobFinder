TÀI LIỆU MÃ BIỂU ĐỒ TỪ SƠ ĐỒ USE CASE ĐỒ ÁN
Nguồn chính:
- Sơ đồ use case bạn gửi trong ảnh.
- Logic đăng nhập/phân quyền được chuẩn hóa theo repo hiện tại: AuthRoute, ProtectedRoute, axios interceptor, localStorage token, refresh token, có thể có CAPTCHA và 2FA.
Ghi chú sử dụng:
- Đoạn Mermaid dùng để render sequence diagram.
- Đoạn PlantText dùng cú pháp PlantUML activity diagram trên planttext.com.
- Các use case cần đăng nhập đều có bước kiểm tra phiên trước khi thực hiện.
- "Quản lý nhật ký" được hiểu là xem, lọc, tra cứu và xuất nhật ký hoạt động hệ thống.
- "Quản lý bài đăng" được hiểu là kiểm duyệt và quản lý bài đăng nội dung/tuyển dụng ở mục quản trị.
================================================================
UC01 - Đăng ký
[Mermaid]
sequenceDiagram
    actor Guest as Khách vãng lai
    participant UI as Hệ thống(UI)
    participant Auth as Controller
    participant DB as CSDL
    participant Notify as OTP Service
    Guest->>UI: Mở form đăng ký
    Guest->>UI: Nhập email và mật khẩu
    UI->>Auth: Gửi thông tin đăng ký
    Auth->>DB: Kiểm tra email đã tồn tại
    alt Email hợp lệ và chưa tồn tại
        Auth->>DB: Tạo tài khoản mới
        Auth->>Notify: Gửi email xác minh/hướng dẫn đăng nhập
        Auth-->>UI: Trả kết quả thành công
        UI-->>Guest: Thông báo đăng ký thành công và chuyển đến đăng nhập
    else Email đã tồn tại hoặc dữ liệu sai
        Auth-->>UI: Trả thông báo lỗi
        UI-->>Guest: Hiển thị lỗi và yêu cầu nhập lại
    end
[PlantText]
@startuml
skinparam shadowing false
title UC01 - Đăng ký
|Khách vãng lai|
start
:Mở trang đăng ký;
:Nhập email và mật khẩu;
:Nhấn Đăng ký;
|Hệ thống|
:Kiểm tra tính hợp lệ dữ liệu;
:Kiểm tra email đã tồn tại;
if (Hợp lệ?) then (Có)
  :Tạo tài khoản mới;
  :Gửi email xác minh/hướng dẫn;
  |Khách vãng lai|
  :Nhận thông báo đăng ký thành công;
  stop
else (Không)
  |Khách vãng lai|
  :Nhận thông báo lỗi và nhập lại;
  stop
endif
@enduml

UC02 - Đăng nhập
[Mermaid]
sequenceDiagram
    actor User as Người dùng
    participant UI as Hệ thống(UI)
    participant Auth as Controller
    participant DB as CSDL
    participant Session as Dịch vụ phiên
    User->>UI: Nhập tài khoản và mật khẩu
    UI->>Auth: Gửi yêu cầu đăng nhập
    Auth->>DB: Kiểm tra thông tin đăng nhập
    alt Tài khoản hợp lệ
        opt Tài khoản bật 2FA
            Auth-->>UI: Yêu cầu mã 2FA
            User->>UI: Nhập mã 2FA
            UI->>Auth: Xác minh mã 2FA
        end
        Auth->>Session: Tạo phiên, cấp token
        Session-->>Auth: Token/Session hợp lệ
        Auth-->>UI: Đăng nhập thành công
        UI-->>User: Lưu token và chuyển đến trang chủ
    else Sai thông tin hoặc bị khóa
        Auth-->>UI: Trả thông báo lỗi/CAPTCHA
        UI-->>User: Hiển thị lỗi đăng nhập
    end
[PlantText]
@startuml
skinparam shadowing false
title UC02 - Đăng nhập
|Người dùng|
start
:Nhập tài khoản và mật khẩu;
:Nhấn Đăng nhập;
|Hệ thống|
:Kiểm tra thông tin đăng nhập;
if (Thông tin hợp lệ?) then (Có)
  if (Bật 2FA?) then (Có)
    |Người dùng|
    :Nhập mã 2FA;
    |Hệ thống|
    :Xác minh mã 2FA;
    if (Mã đúng?) then (Có)
      :Tạo phiên và cấp token;
      |Người dùng|
      :Vào hệ thống;
      stop
    else (Không)
      |Người dùng|
      :Nhận thông báo sai mã 2FA;
      stop
    endif
  else (Không)
    :Tạo phiên và cấp token;
    |Người dùng|
    :Vào hệ thống;
    stop
  endif
else (Không)
  :Có thể yêu cầu CAPTCHA hoặc thông báo tài khoản bị khóa;
  |Người dùng|
  :Nhận thông báo lỗi;
  stop
endif
@enduml

UC12 - Ứng tuyển
[Mermaid]
sequenceDiagram
    actor Candidate as Ứng viên
    participant UI as Hệ thống(UI)
    participant Auth as Controller
    participant ApplySvc as Dịch vụ ứng tuyển
    participant CV as Dịch vụ CV
    participant DB as CSDL
    participant Notify as OTP Service
    Candidate->>UI: Nhận Ứng tuyển
    UI->>Auth: Kiểm tra đăng nhập
    alt Chưa đăng nhập
        Auth-->>UI: Phiên không hợp lệ
        UI-->>Candidate: Chuyển đến trang đăng nhập
    else Đã đăng nhập
        UI->>CV: Kiểm tra CV/mặc định hồ sơ ứng tuyển
        CV->>DB: Lấy CV mặc định
        DB-->>CV: Thông tin CV
        alt Hồ sơ/CV hợp lệ
            Candidate->>UI: Xác nhận thông tin ứng tuyển
            UI->>ApplySvc: Gửi hồ sơ ứng tuyển
            ApplySvc->>DB: Tạo bản ghi ứng tuyển
            ApplySvc->>Notify: Thông báo cho nhà tuyển dụng
            DB-->>ApplySvc: Tạo thành công
            ApplySvc-->>UI: Trả kết quả thành công
            UI-->>Candidate: Hiển thị đã ứng tuyển thành công
        else Chưa đủ hồ sơ
            CV-->>UI: Thiếu CV/thông tin
            UI-->>Candidate: Yêu cầu bổ sung hồ sơ
        end
    end
[PlantText]
@startuml
skinparam shadowing false
title UC12 - Ứng tuyển

start

if (Đã đăng nhập?) then (Có)
  :Kiểm tra CV và hồ sơ mặc định;
  if (Hồ sơ đầy đủ?) then (Có)
    |Ứng viên|
    :Xác nhận thông tin ứng tuyển;
    |Hệ thống|
    :Tạo hồ sơ ứng tuyển;
    :Gửi thông báo cho nhà tuyển dụng;
|Ứng viên|
:Nhận thông báo thành công;
stop
  else (Không)
    |Ứng viên|
    :Bổ sung CV/hồ sơ;
    stop
  endif
else (Không)
  |Ứng viên|
  :Đăng nhập để tiếp tục;
  stop
endif
@enduml

PROMPT GỢI Ý ĐỂ GỬI CHO COPILOT TRONG VS CODE
Hãy mở file `usecase_diagrams_do_an_ordered.txt` trong workspace hiện tại và thực hiện đúng các yêu cầu sau:
1. Đọc toàn bộ 30 use case trong file.
2. Kiểm tra syntax của từng đoạn Mermaid và PlantUML, nếu có lỗi render thì sửa syntax nhưng không đổi tên use case và không đổi logic nghiệp vụ.
3. Tách mỗi use case thành 2 file riêng:
   - `docs/sequence/ucxx-ten-use-case.mmd`
   - `docs/activity/ucxx-ten-use-case.puml`
4. Tạo thêm `docs/README_usecase_diagrams.md` gồm:
   - danh sách 30 use case
   - đường dẫn tới file `.mmd` và `.puml`
   - hướng dẫn render Mermaid và PlantText ngắn gọn
5. Giữ nguyên ngôn ngữ tiếng Việt cho tên use case trong nội dung biểu đồ.
6. Nếu thay một use case trong file bị mơ hồ, hãy thêm comment ngắn ở đầu file đó để ghi rõ giả định nghiệp vụ, nhưng không được tự ý bỏ use case nào.
7. Sau khi hoàn thành, trả về bảng tổng hợp gồm: tên use case, file Mermaid, file PlantUML, ghi chú nếu đã sửa syntax.
Nếu cần tối ưu hóa, ưu tiên:
- Sequence diagram ngắn gọn, dễ đọc, có `alt/else` cho nhanh quyết định.
- Activity diagram có swimlane `|Người dùng|` và `|Hệ thống|` hoặc actor tương ứng.
- Không sử dụng ký tự đặc biệt để tránh lỗi render nếu không cần thiết.
